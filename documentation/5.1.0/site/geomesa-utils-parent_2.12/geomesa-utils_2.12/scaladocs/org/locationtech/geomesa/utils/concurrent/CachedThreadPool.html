CachedThreadPool 

Companion object CachedThreadPool

class CachedThreadPool extends AbstractExecutorService with LazyLogging

Executor service that will grow up to the number of threads specified.

Assertion: core java executors offer 3 features that we want, but you can only use two of them at once:

  1. Add threads as needed but re-use idle threads if available 2. Limit the total number of threads available 3. Accept and queue tasks once the thread limit is reached

This executor allows us to do all three. It is backed by an unlimited, cached thread pool that will handle re-use and expiration of threads, while it tracks the number of threads used. It is fairly lightweight, and can be instantiated for a given task and then shutdown after use.
