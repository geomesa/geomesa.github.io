27 /**
28   * Version 2 serialization scheme consists of:
29   *
30   * <ol>
31   *   <li>one byte containing the serialization version (2)</li>
32   *   <li>four byte int containing the offset to the per-attribute offsets</li>
33   *   <li>the feature id as a string (if `withId`)</li>
34   *   <li>the serialized attributes, in order, each prepended with a byte 0 or 1 to indicate null/not null</li>
35   *   <li>a variable length int (1-4 bytes) per attribute, for the offset to the start of the attribute in the serialized bytes</li>
36   *   <li>the serialized user data (if `withUserData`)</li>
37   * </ol>
38   */
